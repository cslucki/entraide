### ⚠️ WARNING: Missing community_id indexes (critical for prod scale)
- `organization_id` has indexes on all 5 tables (from migration)
- `community_id` has **NO indexes** on any table — all tenant-scoped queries filter on `community_id`, causing sequential scans at scale
- **RISK**: Production datasets will degrade. Recommend adding `community_id` indexes matching the existing `organization_id` index pattern
- **SEVERITY: MEDIUM** (not blocking for mirror, but affects perf)

### ⚠️ WARNING: LIKE case sensitivity on PostgreSQL (7 files, ~16 queries)
TASK-062 fixed `SearchController.php` with driver-aware `like`/`ilike`. Still unfixed:

| File | Lines | Search Targets |
|------|-------|----------------|
| `app/Livewire/Explorer.php` | 139-141, 194 | title, description, tags.name |
| `app/Http/Controllers/Api/ServiceController.php` | 19 | title, description |
| `app/Http/Controllers/Api/ServiceRequestController.php` | 19 | title, description |
| `app/Http/Controllers/Admin/AdminController.php` | 53-54, 242, 351-352, 372 | name, email, title, buyer/seller name |
| `app/Http/Controllers/Admin/AdminBlogController.php` | 19-20 | title, content |
| `app/Http/Controllers/Admin/AdminMessageController.php` | 19, 33-35 | body, sender/email, buyer/email, seller/email |
| `app/Http/Controllers/Admin/AdminEmailTemplatesController.php` | 16-17 | name, slug |
| `app/Http/Controllers/Admin/AdminEmailLogsController.php` | 19-20 | to_email, subject |

**Impact on PostgreSQL**: `LIKE` is case-sensitive → search misses uppercase/lowercase matches
**Impact on SQLite**: `LIKE` is case-insensitive → different behavior between runtimes
**Fix**: Extract `$likeOperator = DB::connection()->getDriverName() === 'pgsql' ? 'ilike' : 'like'` or create a global helper/macro
**SEVERITY**: Medium — behavioral diff between SQLite dev and PostgreSQL prod

### ⚠️ WARNING: DB::raw() points_balance — SQL injection risk
- `TransactionController.php:198-199` and `Api/TransactionController.php:201-202` concatenate `$points` directly into `DB::raw()`
- `$points` comes from `$tx->points_agreed` (DB-sourced integer), so injection is unlikely but pattern is fragile
- **SEVERITY**: Low — safe today but should use parameterized `DB::update()` or Eloquent `increment()/decrement()`

### ℹ️ INFO: Reserved slug regex overlap
- `$communityConstraint` uses a negative lookahead for reserved words: `login|register|admin|api|sitemap|search|explorer|profile|password|membres|echanges|boucles`
- `$organizationConstraint` duplicates this pattern for future `/org/{organization}` routes
- **STATUS**: Correct — no change needed
