# Review Notes

## Implementation choices

- URL referral : `/{community}/register?ref={code}` — route legacy existante, pas de nouvelle route
- Card placement : après raccourcis secondaires, évite tout risque de casse layout
- Copie : Alpine + navigator.clipboard.writeText (Alpine déjà présent dans le projet)
- Pas de WhatsApp button — trop spécifique, serait mieux dans une itération suivante
- Try/catch silencieux sur ref invalide — la registration ne doit jamais blocker
- Cross-org ref : ReferralService rejette déjà avec RuntimeException → catché silencieusement

## Risques restants

- Le lien URL contient le slug Community dans le path (legacy technique) — pas de wording "Community" côté UI
- Le wording "boucle" est utilisé ("faire entrer dans la boucle") — cohérent produit
- Cross-organization non testé end-to-end dans ReferralRegistrationTest (couvert par ReferralServiceTest)
