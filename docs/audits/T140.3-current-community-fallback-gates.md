## Architecture actuelle (binding chain)

```
WEB REQUEST
  │
  ├─ 1. ResolveUrlOrganization (web group middleware)
  │     └─ bindOrganization():
  │          ├─ current_organization = Community (toujours)
  │          └─ current_community = Community (si pas déjà bindé)
  │
  └─ 2. ResolveCommunity (route middleware /{community}/...)
        └─ handle():
             ├─ current_community = Community (toujours)
             ├─ current_organization = Community (toujours)
             └─ View::share('currentCommunity', ...)
             └─ View::share('currentOrganization', ...)

API REQUEST
  │
  └─ 1. ResolveApiOrganization (api group middleware)
        └─ bindOrganization():
             └─ current_organization = Organization (toujours)
             └─ (current_community JAMAIS bindé - déjà clean)
```

## Résolution runtime

```
CurrentOrganization::get()
  ├─ 1. app('current_organization')     ← canonique
  ├─ 2. app('current_community')         ← fallback legacy (T140.3)
  └─ 3. null                              ← fail-closed
```

## Usages intentionnels à garder

- `ResolveCommunity` line 22 : binding double (commun/org) sur routes `/{community}`
- `ResolveUrlOrganization` lines 250-251 : binding conditionnel pour compat routes legacy
- `CurrentOrganization::get()` lines 29-30 : fallback documenté

## Usages dangereux (suppression future)

- **currentCommunity view share** (`ResolveCommunity` lines 24, 27) : crée une dépendance UI à une variable legacy redondante
- **$currentCommunity dans Blade** : dead code mais crée un précédent d'usage

## Plan de retrait futur

Ordre de suppression recommandé (non exécuté dans T140.3) :

### T140.4 — Routes /org/{organization} en parallèle
- Créer le groupe de routes `/org/{organization}` avec middleware `organization`
- Garder les routes `/{community}` existantes
- Tests de cohabitation

### T140.5 — Migration controllers/services/API/channels
- Supprimer `current_community` binding de `ResolveUrlOrganization::bindOrganization()`
- Migrer `ResolveApiOrganization` vers `organization_id` comme source
- Migrer les broadcast channels
- Adapter les tests cassés (listés ci-dessus)

### T140.6 — Migration tests et vues
- Retirer `View::share('currentCommunity', ...)` de `ResolveCommunity`
- Nettoyer `$currentCommunity` des vues Blade
- Adapter les tests de compatibilité
- Supprimer known-risks résolus

### T140.8 — Dépréciation routes /{community}
- Redirect `/{community}` → `/org/{organization}`
- Annoncer dépréciation

### T140.9 — Retrait final
- Supprimer `current_community` fallback de `CurrentOrganization::get()`
- Supprimer `Community.php` (ou le réduire à un alias minimal)
- Supprimer colonnes legacy (`community_id` sur users, etc.)
- Rename table `communities` → `organizations`
